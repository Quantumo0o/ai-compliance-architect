## 💎 Key Production Features (Local-Only)

### 1. Zero-Dependency Scalability
Unlike enterprise apps that require Postgres or Redis, this system is **100% Portable**. It uses a high-performance **SQLite Write-Ahead Logging (WAL)** mode and an **Asynchronous Database (DB) Write Queue** to handle 25+ concurrent analysis tasks without database locking errors.

### 2. Failure-Resistant Persistence
Using a dedicated `task_queue` persistence table, the system is immune to server crashes. If the FastAPI backend restarts mid-analysis, it automatically resumes all "In Progress" adjudications on startup.

### 3. Page-Stitching Data Integrity
To solve the industry-wide "Page Split" problem, the system uses a **Page Stitching Buffer**. It merges overlapping context between pages to ensure that large multi-page legal requirements are extracted as a single, coherent clause.

### 4. Asynchronous Double-Blind Adjudication
To eliminate "confirmation bias," the system uses two isolated RAG paths. The **Critic Agent** performs its own independent search from a private vector index to challenge the **Primary Judge** on high-risk compliance items.

## 🛠️ How it Works: Step-by-Step

### Step 1: Initialize Knowledge Base
Populate `data/knowledge_base/` and run the vector ingestion to create your local ChromaDB search index.

### Step 2: Launch the Processes
1.  **Backend Application Programming Interface (API)**: `uvicorn src.api.server:app --port 8000`
2.  **Frontend User Interface (UI)**: `streamlit run app.py`

### Step 3: Fast Ingestion & Recovery
Upload your Portable Document Format (PDF) Request for Proposal (RFP). The system uses **Risk Score Heuristics** to intelligently trigger Optical Character Recognition (OCR) only when needed, and initializes a persistent task record so your progress is never lost.

### Step 4: Requirement Extraction
`mistralai/mistral-large-3-675b-instruct-2512` extracts structured requirements using the **Page Stitching** buffer for 100% capture rate.

### Step 5: Asynchronous Review & Conflict Detection
The FastAPI worker parallelizes the work. Conflicts between the Judge and Critic are instantly flagged in the User Interface (UI) for human review.

### Step 6: Commit & Export
Correct any findings in the matrix and click **💾 Commit Changes**. Export your final, audited matrix to CSV or Excel.

### 2. Domain-Segregated Reranking (The "Tender Blindness" Fix)
To prevent the company knowledge from "drowning out" the client's RFP details during chat retrieval, we implemented a mathematically hard-balanced RAG pipeline:
*   **Isolated Pool Retrieval**: Company facts and RFP facts are retrieved and reranked in two totally separate buckets.
*   **50/50 Multi-Source Weaving**: The system forces exactly 5 facts from the Company Knowledge and 5 facts from the RFP into the Chatbot's context window.
*   **Conflict Resolution**: This ensures the AI always has the "Rules of the Tender" in its immediate memory, even if the "Company Profile" is semantically stronger.
